What is this?

This a simple snake like game rendered using Autostereogram. I recently rediscovered Autostereograms and after some research realized that I was always viewing them uing cross pattern, which is incorrect method. After some research I found a paper that had algorithm for stereogram generation in C, I decided to try to duplicate this algorithm to see if it would be possible to render moving objects using this technique. Both game and rendering algorithm are implemented in rust and are running on WASM in your browser. Source code can be found here. At the moment my main struggle is the fact that I still can't view them correctly, so validating accuracy of my algorithm is only possible with outside help, which slows things drastically. Also im not sure where is my implementation off, but people report some "fuzzines" around the endegs which at the moment I'm not sure how to fix.

Big thanks to author of MagicEye.js library which I used to compare my implementation against and used as a base for my canvas manipulation code.
